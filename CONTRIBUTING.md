## Commit Message Guidelines

```bash
$ git commit -m '<Prefix>: Message here...'
```

下記のいずれかの`Prefix`をコミットメッセージに含めてください。

- chore: プロダクションコードに変更を加えない作業
- docs: ドキュメントの修正、更新
- feat: 新機能の実装
- fix: バグ修正
- perf: パフォーマンスを向上させるコードの変更
- refactor: バグ修正や機能追加ではないコードの変更
- style: 空白、書式設定、セミコロンの欠落など
- test: テストの追加、修正
